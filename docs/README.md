# Index de documentation

Ce dossier contient la documentation technique du projet ObRail.

## A lire en premier
- `docs/architecture.md` pour la vue d'ensemble et les composants
- `docs/setup.md` pour lancer la stack en local
- `docs/flow.md` pour le flux ETL et la correspondance avec le cahier des charges
- `docs/data.md` pour la structure des donnees et le schema
- `docs/api.md` pour les endpoints API
- `docs/runbook.md` pour l'exploitation et le depannage
- `docs/testing.md` pour les tests
- `docs/decisions.md` pour les decisions de conception

## Conventions
- Garder la documentation en ASCII si possible.
- Mettre a jour `docs/flow.md` si le flux ETL change.
- Mettre a jour `docs/api.md` si les endpoints changent.
