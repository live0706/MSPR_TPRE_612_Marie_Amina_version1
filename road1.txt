PROGRAMME DE FORMATION DÉVELOPPEUR EN INTELLIGENCE ARTIFICIELLE


CERTIFICATION PROFESSIONNELLE RNCP36581 

Informations Générales

Bloc E6.1 : Créer un modèle de données d'une solution I.A en utilisant des méthodes de Data science.


Objet : Cahier des Charges de la MSPR « Mise en œuvre d'un processus ETL répondant au besoin d'une solution I.A ».

Compétences Évaluées (Certification RNCP36581)

Collecte de données : Définir les sources et les outils nécessaires pour la collecte.


Recueil sécurisé : Extraire des informations de sources hétérogènes (internes ou Open Data) pour l'architecture de données.


Paramétrage : Configurer les outils pour un import automatisé et sécurisé.


Qualité des données : Analyser, nettoyer et trier les données pour les rendre exploitables via la Data Science.


Stockage : Construire le modèle de données optimal pour le besoin d'analyse.


Visualisation : Créer des graphiques et des tableaux de bord accessibles.


Exploitation : Automatiser le requêtage pour répondre aux exigences de la solution IA.

Compétences Spécifiques au Programme

Automatisation : Programmer des scripts pour l'extraction (web scraping, fichiers, bases de données, Big Data).


Requêtage : Développer des requêtes SQL et des méthodes d'extraction adaptées aux systèmes de gestion de bases de données.


Préparation : Programmer l'agrégation de données, la suppression des entrées corrompues et l'homogénéisation des formats.


Conformité : Créer des bases de données respectant le RGPD avec des modèles conceptuels et physiques (MCD/MPD).

Environnement et Outils (Extraits)
Langages : Python (pandas, requests, SQLAlchemy), R (tidyverse), SQL.

Outils : Docker (conteneurisation), Postman (test d'API).

Bases de données : PostgreSQL.

Frameworks API : Flask, FastAPI ou Node.js.

Sources de données suggérées
Transitland : Jeux de données GTFS (transport public).

Husahuc : Données GTFS France.

Back-on-Track : Base de données sur les trains de nuit en Europe.
